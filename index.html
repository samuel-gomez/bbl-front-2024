Pseudo-element :has()

Le pseudo-élément :has() est une fonction CSS qui permet de sélectionner un élément qui contient un autre élément qui correspond à un sélecteur donné.

                 selecteur:has(selecteur)

Cas basique

On va donc pouvoir sélectionner un élément parent en fonction de la présence d'un élément enfant.

                 .card {
  /* style de base */
}
.card:has(img) {
  /* style de surcharge */
}

Custom layout

Si on peut déjà faire des choses intéressantes avec le cas basique, on peut aller plus loin en utilisant le pseudo-élément :has() pour créer des layouts personnalisés.

                 .card:has(h2 + img + p + p) {
  grid-template-areas: 
    "img img"
    "h2 h2"
    "p1 p2"
}
.card:has(h2 + img + img + p + p) {
  grid-template-areas: 
    "h2 h2"
    "img1 img2"
    "p1 p2"
}

ET logique

Il est possible de chainer l'utilisation du pseudo-élément :has() afin d'avoir un ciblage plus précis.

Ici, on cible une carte qui possède une image ET un titre. 
                 .card:has(img):has(h2) {
  /* style */
}

ET/OU logique

Si on souhaite un ciblage plus souple, on peut fournir une liste de sélecteurs à la fonction :has().

Ici, on cible une carte qui possède soit une image, soit un titre. ou les deux. 
                 .card:has(img, h2) {
  /* style */
}

Démo : ET/OU logique

Sélection globale

Pour finir dans les cas classiques du :has(), on peut utiliser la fonction pour sélectionner un élément parent en fonction de la présence d'un élément lointain enfant.

Dans l'exemple ci-dessous, on cherche à cibler le footer si quelque part dans le DOM ma checkbox est cochée. 
                 body:has(input.myCheckBoxTheme:checked) footer {
  /* style */
}

Démo : Sélection globale

Eléments adjacents direct

Avec les combinateurs "+", on peut cibler un élément en fonction de la présence d'un élément précédent.

Combinateur "+"

on cible les paragraphes immédiatement précédés d'un h2. 
                    h2 + p {  
}

has() et combinateur "+"

on cible les h2 immédiatement suivis d'un paragraphe. 
h2:has(+ p) {
}

Démo : Eléments adjacents direct

Eléments adjacents indirect

Avec les combinateurs "~", on peut cibler un élément en fonction de la présence d'un élément précédent direct ou indirect.

Combinateur "~"

on cible les paragraphes précédés d'un h2 direct ou indirect 
h2 ~ p {
}

has() et combinateur "~"

on cible les h2 suivis d'un paragraphe direct ou indirect. 
h2:has(~ p) {
}

Démo : Eléments adjacents indirect

Cibler un intervalle

On peut également cibler un intervalle d'éléments adjacents

Dans l'exemple ci-dessous, on cherche à cibler le footer si quelque part dans le DOM ma checkbox est cochée. 
                 .from ~ :has(~ .to) {
}

Démo : Cibler un intervalle

Ciblage par quantité

On peut appliquer un style en fonction de la quantité d'éléments ciblés.

Voir la documentationDans l'exemple ci-dessous, on cible les listes qui possèdent au moins 5 éléments. 
                 ul:has(> *:nth-child(5n)) {
}

Ciblage par attribut

On peut appliquer un style en fonction de la valeur d'un attribut.

On peut tirer partie de la puissance du sélecteur d'attribut et ses "expressions régulières". 
                 &:has(a[href$=".wave"]) {
  --theme-color: purple;
  --theme-icon: "🔈";
  --theme-file: " Audio";
}
&:has(a[href$=".jpg"]) {
  --theme-color: sienna;
  --theme-icon: "🏞";
  --theme-file: " Image";
}
&:has(a[href$=".pdf"]) {
  --theme-color: darkred;
  --theme-icon: "PDF";
  --theme-file: " PDF";
}
&:has(a[href$=".docx"]) {
  --theme-color: blue;
  --theme-icon: "DOCX";
  --theme-file: " DOCX";
}
&:has(a[href$=".xslx"]) {
  --theme-color: darkgreen;
  --theme-icon: "XLSX";
  --theme-file: " XLSX";
}

export const usePeople = () => {
  const { data, isLoading, error, refetch } = useQuery({
    queryKey: [ENDPOINT],
    select: ({ responseBody }: TPeopleDataResponse) => ({
      anomaly: setAnomalyEmptyItems(responseBody),
      [SERVICE_NAME]: computeInfos(responseBody),
    }),
  });

  return {
    ...data,
    anomaly: (error || data?.anomaly) as Tanomaly | null,
    isLoading,
    refetch,
  };
};
